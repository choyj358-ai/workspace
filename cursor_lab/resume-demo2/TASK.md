## 개발 임무 (TASK)

### 1단계: 기본 뼈대 + 레이아웃 구성

- **1-1. 프로젝트 기본 파일 생성**
  - `index.html` 파일 생성
  - `style.css` 파일 생성
  - `index.html`에서 `style.css`를 `<link rel="stylesheet" href="style.css">` 로 연결

- **1-2. 전체 레이아웃 그리드 설정**
  - `index.html`에 전체 페이지를 감싸는 최상위 컨테이너 `div` 생성 (예: `.page-grid`)
  - CSS에서 `.page-grid`에 `display: grid` 설정
  - Grid를 이용해 큰 영역을 섹션 단위로 위에서 아래로 배치할 기본 레이아웃 정의

- **1-3. 섹션별 div 컴포넌트 뼈대 만들기**
  - 아래 섹션들을 모두 `div`로 만들고, 각 섹션에 클래스와 id를 부여
    - 헤더 / 히어로 영역: `div.header` (또는 `#header`)
    - 내 정보 섹션: `div.about`
    - 내 좌우명 섹션: `div.motto`
    - 기술 스킬 섹션: `div.skills`
    - 자격증 섹션: `div.certificates`
    - 포트폴리오 섹션: `div.portfolio`
    - 푸터 섹션: `div.footer`
  - 이 단계에서는 **텍스트 내용은 임시 텍스트(placeholder)** 로만 넣고, 구조만 잡는다.

- **1-4. 섹션 내부 구조용 div 나누기**
  - 각 섹션 안에서도 컴포넌트 단위로 `div`를 나눈다. 예:
    - 공통 섹션 제목: `div.section-title`
    - 내용 컨테이너: `div.section-content`
    - 포트폴리오 카드: `div.portfolio-card`
  - 이 역시 구체적인 내용보다, **어디에 무엇이 올지 위치를 잡는 목적**으로만 작성한다.

- **1-5. 개발용 시각 가이드 (padding + border) 적용**
  - `style.css`에서 **모든 `div`에** 아래 스타일을 임시로 적용한다.
    - `padding: 5px;`
    - `border: 1px solid black;`
  - 이 단계에서의 목표:
    - HTML 구조와 Grid 배치만으로 전체 레이아웃과 영역 위치를 눈으로 확인할 수 있게 만든다.

---

### 2단계: 섹션별 레이아웃/콘텐츠 구체화 (PRD 기반)

- **2-1. 헤더 / 히어로 영역 상세화**
  - 이름, 직무 타이틀, 한 줄 소개, 핵심 키워드(마이크로서비스 / AI / Flutter / MVVM)를 위한 요소들을 `div`로 세분화.
  - Grid를 사용해 텍스트 정렬 및 배치 구조를 정돈.

- **2-2. 내 정보 섹션 상세화**
  - 이름, 이메일, GitHub/LinkedIn 링크, 간단 자기소개 문단을 위한 `div` 구조 설계.
  - 연락처/링크는 나중에 실제 값으로 교체할 수 있도록 placeholder 텍스트로 먼저 구성.

- **2-3. 내 좌우명 섹션 상세화**
  - 좌우명 문구와 설명 문단을 분리된 `div`로 구성.
  - 좌우명 텍스트가 시각적으로 강조될 수 있도록, 나중에 스타일을 주기 쉬운 구조로 설계.

- **2-4. 기술 스킬 섹션 상세화**
  - Backend/Microservices, AI/ML/LLM, Mobile & Flutter, Architecture(MVVM), 기타 툴 등 카테고리별 `div` 생성.
  - 각 카테고리 안에 스킬 아이템 리스트를 담는 구조 설계 (스킬 이름 + 숙련도 표시용 요소).
  - Grid를 사용해 카테고리/아이템을 보기 좋게 배치.

- **2-5. 자격증 섹션 상세화**
  - 자격증 목록을 표시할 리스트/카드용 `div` 구조 설계 (자격증 이름, 발급기관, 취득연도).
  - 최신 취득 순으로 정렬하기 쉽게 위에서 아래로 나열.

- **2-6. 포트폴리오 섹션 상세화**
  - 프로젝트 카드를 위한 `div.portfolio-card` 구조 정의:
    - 프로젝트 이름
    - 한 줄 설명
    - 사용 기술 태그
    - 역할/기여도
    - 링크 버튼 영역 (GitHub / Demo)
  - Grid를 사용해 여러 프로젝트 카드가 정렬되도록 배치.

- **2-7. 푸터 섹션 상세화**
  - 이메일, GitHub, LinkedIn 등의 연락처를 담는 `div` 구성.
  - 간단한 문구(예: “협업을 기다리고 있습니다.”)를 표시할 영역 추가.

### 3단계: 디자인 정제 (HTML/CSS, RULES.md 준수)

- **3-1. 색상 정의**
  - `RULES.md` 기준에 따라 **최대 4개의 색상 팔레트**를 정한다.  
    (배경색, 메인 포인트 색, 서브 포인트 색, 텍스트 색)
  - `style.css` 상단에 색상 관련 CSS 변수를 정의해 두면 유지보수에 용이.

- **3-2. 섹션별 기본 스타일 적용**
  - 각 섹션(`.header`, `.about`, `.motto`, `.skills`, `.certificates`, `.portfolio`, `.footer`)에 적당한 여백, 정렬, 타이포그래피 스타일 적용.
  - 여전히 Grid를 중심으로 레이아웃을 유지.

- **3-3. 컴포넌트(카드, 섹션 타이틀) 스타일링**
  - `div.section-title`, `div.section-content`, `div.portfolio-card` 등에 공통 디자인 패턴 적용.
  - 포트폴리오 카드에 hover 등 간단한 시각 효과를 CSS로만 추가 가능.

- **3-4. 반응형 레이아웃 조정**
  - CSS Grid의 `auto-fit`, `minmax` 등을 이용해 화면 크기별 레이아웃 변경.
  - 모바일에서 세로 스크롤 기반으로 모든 섹션이 잘 보이는지 확인.

---

### 4단계: 실제 콘텐츠 입력 및 마무리

- **4-1. 실제 이력/포트폴리오 내용 채우기**
  - PRD에서 정의한 항목에 맞춰 실제 텍스트(자기소개, 좌우명, 스킬 리스트, 자격증, 프로젝트 내용 등)를 입력.

- **4-2. 링크/연락처 실제 값 적용**
  - 이메일 `mailto:` 링크, GitHub/LinkedIn URL, 포트폴리오 GitHub/Demo 링크를 실제 값으로 교체.

- **4-3. 최종 점검**
  - `RULES.md`와 `PRD.md` 규칙/요구사항을 모두 만족하는지 확인.
  - JavaScript/서버/DB/React 등이 사용되지 않았는지 다시 확인.
  - 레이아웃이 Grid 기반인지, 색상이 4개 이하로 유지되는지 점검.
