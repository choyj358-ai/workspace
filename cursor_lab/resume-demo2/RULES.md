## 개발 규칙 (Resume 사이트)

- **기술 스택**

  - **HTML/CSS만 사용**한다.
  - **JavaScript, DB, 서버 사이드 기술은 사용하지 않는다.**
  - **React 등 모든 프론트엔드 프레임워크/라이브러리는 사용하지 않는다.**

- **파일 구조**

  - 페이지는 `index.html` 하나로 구성한다.
  - 스타일은 별도의 `style.css` 파일로 분리한다.
  - `index.html`에서는 `<link rel="stylesheet" href="style.css">` 로만 CSS를 불러온다.

- **레이아웃 / 배치**

  - 전체 페이지 레이아웃은 **CSS Grid Layout만 사용**해서 구현한다.
  - 주요 섹션(내 정보, 좌우명, 기술 스킬, 자격증, 포트폴리오 등)도 가능하면 Grid 기반으로 배치한다.
  - Flexbox, float, position 기반 레이아웃(절대 좌표 배치 등)은 레이아웃 용도로 사용하지 않는다.
    - 단, 아주 간단한 정렬(텍스트 중앙 정렬 등)을 위한 속성(`text-align` 등)은 허용.

- **색상 규칙**

  - 페이지 전체에서 사용하는 **색상은 최대 4개까지**로 제한한다.
    - 예: 기본 배경색 1개, 메인 포인트 컬러 1개, 서브 포인트 컬러 1개, 텍스트 색 1개.
  - 같은 색의 명도/채도만 살짝 다르게 하는 정도는 허용하되, 완전히 다른 색 계열을 추가하지 않는다.

- **컴포넌트 구조**

  - 페이지는 **컴포넌트 단위의 `div` 박스**로 나누어 개발한다.
    - 예: `header`, `about`, `motto`, `skills`, `certificates`, `portfolio`, `footer` 등 섹션별 컨테이너 `div`.
  - 각 컴포넌트는 **의미 있는 클래스 이름**을 가진 `div`를 기준으로 스타일링한다.
    - 예: `.section-header`, `.section-skills`, `.card-portfolio` 등.
  - CSS에서는 이 컴포넌트 클래스 단위로 스타일을 정의하고, 불필요하게 복잡한 선택자는 사용하지 않는다.

- **반응형 / 기타**

  - 가능하면 CSS Grid의 `auto-fit`, `minmax` 등을 활용해 **간단한 반응형**을 구성한다.
  - 애니메이션/트랜지션이 필요하면 CSS만으로 구현하며, 과도한 효과는 피한다.

- **금지 사항 정리**
  - JavaScript 코드 작성 금지.
  - 서버 통신, API 호출, DB 연동 금지.
  - React, Vue, Svelte, Angular 등 모든 프론트엔드 프레임워크/라이브러리 사용 금지.
  - 외부 CSS 프레임워크(Bootstrap, Tailwind 등) 사용 금지. (순수 CSS만 사용)
